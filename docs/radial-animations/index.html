radial chart animations

How Le Jardin Radial animates ring transitions — and why each animation exists. Every decision solves a specific visual problem.

1. the gap problem

A radial chart has concentric rings. When flowers are added, removed, or reordered, the rings change position. The naive approach — independently animating each ring from old position to new — creates visible gaps between rings.

why gaps appear

When a ring is removed, it collapses to its midpoint. But the remaining rings shift to fill the space at a different rate. For a brief moment, no ring covers the area between them.

3 rings, no gaps
→
gap mid-transition

2. removing a flower

the fix: boundary-aligned collapse

Instead of collapsing to the ring's midpoint, the exiting ring collapses to the exact position where its neighbors will meet in the new layout. This keeps ring edges perfectly contiguous at every frame of the animation.

Exception: the innermost ring still collapses to its center — the familiar "shrink to center" visual feels more natural there, and the adjacent ring's edge naturally overlaps it.
collapse to boundary, not midpoint
→
no gap contiguous

3. adding a flower

A new flower expands from a zero-width line at the boundary between its future neighbors. This is gap-free by construction — the expansion point is exactly where the adjacent rings meet.

expands from the boundary between neighbors

4. reordering — the hero ring

When a flower is dragged to a new position, the animation faces two challenges:

  • Rings passing through each other would leave gaps (same problem as removal)
  • The user needs to visually track which flower they're moving

the solution: hero + background circles

The dragged flower (the "hero") stays as a visible ring and moves to a foreground SVG layer — above the curved text labels. Everything else becomes filled circles (extending from the center), providing a seamless gap-free background.

Z-order: circles → labels → hero ring
The hero ring physically covers labels as it slides past them — no opacity tricks, just real SVG layering.

direction-aware easing

The hero and the background rings don't move at the same speed. Each direction gets its own easing curve:

outward (hero)
ease-out — fast launch, gentle settle
inward (others)
ease-in — gentle start, quick finish
The outward flower races ahead. The inward flowers wait, then catch up. The visual effect: the hero "pushes" through while others fill in behind.

5. bulk changes

Reset, import, or saving a shared garden changes many flowers at once. Per-element animations would be chaotic — multiple rings collapsing and expanding simultaneously. Instead, we use a phased cross-dissolve.

old rings
0ms 450ms
survivors slide
200ms 600ms
new rings
600ms 1050ms
labels
800ms 1200ms
Each phase waits for the previous one. The eye follows: old dissolves → survivors settle → new materializes → labels appear.
Intent detection: bulk dissolve only fires when the app explicitly signals it (reset, import, save). Rapid individual clicks get per-element animation — never the dissolve.

6. smooth interruption

If the user clicks rapidly, a new animation starts while the previous one is still running. The naive approach: each ring caches its target position at the start of the animation. If interrupted, the next animation starts from the target — not from where the ring actually is. This causes a visible snap.

__prev = target jumps on interrupt
__prev = current updates every frame

The fix: update __prev on every animation frame, not just at the start. Interrupted transitions pick up from the current visual position.

7. the empty state

When all flowers are removed, ghost ring outlines appear as a skeleton preview of what the chart becomes. The "start planting" button opens the garden panel.

Ghost rings appear with the chart structure (immediately on first load, or after the exit animation when removing all flowers). The text and inner ghost ring fade in with a 500ms delay — waiting for exit animations to complete before claiming the space.

summary

scenario animation why
add flower expand from boundary gap-free by construction
remove (middle) collapse to boundary stays contiguous with neighbors
remove (innermost) shrink to center natural "disappear" visual
reorder hero ring + bg circles gap-free, trackable, label occlusion
reset / import phased cross-dissolve avoids chaotic per-element noise
rapid clicks smooth interruption no snapping to stale targets
